Техническое описание и инструкция по пользованию

- **Операционная система**: Kubuntu 24.04.4 LTS
- **Языки и среды**: Python, Java, C/C++ (gcc, gdb, g++, make), C# (.NET Core), Go, Rust, Node.js
- **Редакторы и IDE**: VS Code, Vim/Neovim
- **Базы данных**: PostgreSQL, SQLite

- **Операционная система**: Ubuntu.
- **Свободное место**: не менее 15 ГБ на диске.
- **Права sudo**: для монтирования, установки пакетов и работы с chroot.
- **Интернет**: для скачивания оригинального ISO и установки пакетов.

- **Операционная система**: Windows 10/11
- **Свободное место**: не менее 50ГБ на диске.
- **ПО**: QEMU, Win32DiskImager

# Инструкция по пользованию:

## Создание кастомного ISO

Для создания файла ОС с дополнительным ПО следуйте инструкции:
1. Клонируйте этот репозиторий на свою систему с Ubuntu:
```bash
git clone https://github.com/ChocolateCake4258/linux-flashcard.git
```
2. Пропишите эту команду в консоль, чтобы сделать все скрипты исполняемыми:
```bash
chmod +x build.sh cleanup.sh config.sh install-chroot.sh prepare.sh
```
3. Запустите скрипт build.sh:
```bash
./build.sh
```
4. Ждите (довольно долгое время), при появлении в консоли вопросов об установке - отвечайте yes
5. При успешном завершении процесса появится требуемый файл - custom-kubuntu.iso

При необходимости версию изначального ISO файла, а также название генерируемого файла можно поменять в файле config.sh

## Создание загрузочного носителя

На этом этапе Вам потребуется ISO файл с предыдущего этапа, ПК с ОС Windows 10/11, QEMU и Win32DiskImager (ссылки на это ПО можно найти в файле BUILD_INFO.md)

1. Клонируйте репозиторий
2. Скопируйте в него папку qemu
3. Создайте виртуальный жесткий диск с помощью скрипта create-virtual-disk.bat (Указывайте объем меньше реального объема физического носителя)
4. Запустите скрипт setup.bat
5. Установите ОС
6. Запустите скрипт convert-disk.bat
7. С помощью Win32DiskImager переместите содержимое виртуального диска на загрузочный носитель

## Использование загрузочного носителя

Вариант 1 - нативное использование через BIOS/UEFI
1. Выберите в BIOS/UEFI загрузочный носитель как загрузчик
2. Сохраните изменения и выйдите
3. Компьютер перезапустится и запустится с Вашего носителя

Вариант 2 - виртуальная машина

1. С помощью утилиты diskpart узнайте физический номер вашего загрузочного носителя
2. Активируйте скрипт start-system.bat от имени администратора
